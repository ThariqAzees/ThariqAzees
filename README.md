## Featured Projects

<details open>
<summary><b>🧠 Webcam-Based ADHD Screening Platform (Hero Project)</b></summary>

<br/>

**Overview**
An AI-powered Human–Computer Interaction research platform for cognitive screening, combining webcam-based gaze tracking with a validated working-memory task and machine learning analysis — built as a full HCI systems paper targeting venues such as IJHCS / Behaviour & IT.

**Problem Statement**
Traditional ADHD screening tools rely on expensive eye-tracking hardware or clinical observation. This project explores whether a standard webcam can enable an accessible, structured cognitive assessment pipeline.

**Architecture**
- Multi-page Streamlit application (Python) with a JavaScript frontend layer
- MediaPipe FaceMesh-based 9-point gaze calibration
- Sternberg working-memory task, replicating a published experimental protocol
- Dedicated Experiment Manager module with SQLite-backed structured event logging
- Feature extraction pipeline feeding a separate ML classification layer

**Technologies**
Python · Streamlit · MediaPipe · JavaScript · SQLite · Scikit-learn

**Features**
- Real-time webcam gaze tracking and calibration
- Cognitive task battery based on peer-reviewed experimental design
- ML classification layer (Random Forest, Gradient Boosting, SVM, XGBoost) with stratified cross-validation
- Structured experiment session management and logging
- Interactive analytics dashboard

**Engineering Challenges**
- Enforcing strict data-pipeline boundaries between modules to preserve experimental validity
- Real-time calibration accuracy using consumer-grade webcams
- Designing a systems architecture rigorous enough to support formal HCI evaluation (usability, performance, robustness, and ML case-study tracks)

**Performance / Scalability**
Modular multi-module architecture (six modules) designed for maintainability and extensibility as evaluation tracks are added.

**Future Improvements**
Full usability study, system performance benchmarking, robustness testing, and completion of the ADHD ML case study as part of the thesis evaluation.

**Status:** Actively under development, targeting submission to an international HCI-focused journal/conference.

**Repository:** [github.com/ThariqAzees/ADHD_Eye_Framework](https://github.com/ThariqAzees/ADHD_Eye_Framework) &nbsp;|&nbsp; **Documentation:** _Coming soon_ &nbsp;|&nbsp; **Live Demo:** _Coming soon_

</details>

<details>
<summary><b>⛏️ Mining Impact Prediction and Visualization System (MIPVS)</b></summary>

<br/>

**Overview**
A seven-module machine learning framework for dam safety assessment under blast-induced ground vibrations from open-pit mining operations.

**Problem Statement**
Open-pit mining blasts near dam infrastructure pose safety risks that are difficult to quantify with traditional threshold-based methods alone.

**Architecture**
- Three physics-based indices: BVII, SDI, MDI
- Random Forest classification layer
- QGIS-based spatial visualization dashboard

**Technologies**
Python · Scikit-learn · QGIS · Pandas · NumPy

**Metrics**
- **94.3%** classification accuracy
- **0.96** AUC
- **0.887** Kappa score
- Evaluated on **1,478** blast records aggregated from published literature, with ground truth derived from ICOLD Bulletin 99 thresholds

**Engineering Challenges**
Integrating physics-based domain indices with a data-driven classifier while maintaining interpretability for a dam-safety engineering audience.

**Status:** Manuscript based on this work is currently **under peer review**.

**Repository:** [github.com/ThariqAzees/MIPVS](https://github.com/ThariqAzees/MIPVS) &nbsp;|&nbsp; **Documentation:** _Available on request_ &nbsp;|&nbsp; **Live Demo:** _N/A (research system)_
